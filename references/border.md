# 分割线 / 边框规范

> 规范层。分割线与边框的**粗细、颜色层级、用法边界**。具体硬值在产物层 [`examples/ui-markdown/token.md`](../examples/ui-markdown/token.md)。

## 粗细阶梯

| token           | 值  | 用途                      |
| --------------- | --- | ------------------------- |
| border-hairline | 0.5 | 极细分隔(列表行/iOS 风格) |
| border-thin     | 1   | 标准分割线/卡片边框       |
| border-medium   | 2   | 强调边框/选中态           |
| border-thick    | 4+  | 容器边界/分隔栏           |

移动端 hairline(0.5px,或 1px @2x)最常见。

## 颜色层级

边框颜色从中性色梯度取,按强制度分层:

- **默认边框** — 中性浅灰(如 gray-200 量级)
- **分割线** — 更浅(灰白)
- **强调/选中边框** — 品牌色
- **危险边框** — danger 色

边框色不引入新色,从中性梯度 + 品牌/功能色复用(见 [color.md](./color.md))。

## 用法边界(慎用分割线)

**大量分割线增加视觉复杂度,破坏整洁感**。分组手段优先级:

1. 用**间距/留白**分组(首选,邻近性,见 [spacing.md](./spacing.md))
2. 用**背景色块**区分区块(卡片底色)
3. 用**分割线**(最后手段,仅当间距 + 底色不足以区分时)

表单场景尤其慎用——通过背景和字段分组减少对分割线的依赖。

## 边框 vs 阴影

卡片容器优先用**阴影**表达层级(轻微 y 偏移,模拟顶光),而非粗边框。边框用于状态(选中/错误),阴影用于深度。

## 一致性

同一视图的分割线粗细 + 颜色 MUST 统一。混用 0.5/1/2px 不同灰度会显脏。
