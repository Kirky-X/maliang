# 字体规范

> 规范层。字体的**层级、行高、字重、字族规则**。具体字号/行高硬值在产物层 [`examples/ui-markdown/token.md`](../examples/ui-markdown/token.md)。

## 字号层级(6 阶)

移动端推荐 6 阶(对应 display / title / subtitle / body / caption / overline):

| 阶  | 角色   | 推荐(pt) |
| --- | ------ | -------- |
| H1  | 大标题 | 24       |
| H2  | 标题   | 18       |
| H3  | 副标题 | 16       |
| H4  | 正文   | 14       |
| H5  | 说明   | 12       |
| H6  | 标注   | 10       |

阶梯原则:相邻阶差 ≥2pt 保证视觉可区分;不超过 6 阶(米勒 7±2,层级过多即失去层级意义)。

## 行高

| 场景        | 行高                 |
| ----------- | -------------------- |
| 标题(≤18pt) | 字号 + 8pt(紧凑)     |
| 正文(≥14pt) | 字号 × 1.4~1.6(舒适) |

标题行高紧凑以突出,正文行高宽松以可读。同一字号的标题与正文用不同行高策略。

## 字重策略

| 字重         | 用途            |
| ------------ | --------------- |
| 400 regular  | 正文            |
| 500 medium   | 强调正文/次标题 |
| 600 semibold | 标题/按钮       |
| 700 bold     | 大标题/数字     |

优先用**字重对比**代替字号对比做层级——避免仅靠字号大小区分,字重 + 字号双维度对比更清晰。

## 字族

- 中文:苹方(PingFang SC)
- 英文/数字:SF Pro / Inter

不超过 2 个字族(性能 + 视觉一致)。中英混排时中文走中文字族,英文/数字走西文字族,数字可单独用等宽或特色字体突出(价格、统计值)。

## 文字透明度梯度(层级靠透明度)

正文基于同一文字色,通过透明度梯度表达层级,而非为每层定义独立颜色:

| 透明度 | 角色 |
| ------ | ---- |
| 100%   | 主文 |
| 80%    | 次文 |
| 60%    | 辅助 |
| 40%    | 占位 |
| 20%    | 装饰 |
| 10%    | 底纹 |

一套透明度梯度统一所有层级,降级时只调透明度不换色。

## 长文本

大段正文颜色适当变浅(避免深色大块文字压迫),但 MUST 保持 WCAG AA(4.5:1)对比度——变浅以不跌破对比度下限为前提。
