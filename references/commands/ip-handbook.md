# ip-handbook 子命令 —— IP 视觉手册全模块产出

> 本文件是 `ip-handbook` 子命令的完整流程,由顶层 [`SKILL.md`](../../SKILL.md) 路由进入。
> 输入 = 项目根目录的 [`DESIGN.md`](./design-md.md) + [`ip`](./ip.md) 子命令产出的 IP 形象 PNG。
> 输出 = **IP 视觉手册**(8 个模块:三视图/动作延展/表情包/字体/色彩/版式/周边/通用要求),落在 `examples/ip-handbook/<project-name>/` 下。
> 这是 [`ip`](./ip.md) 的**下游**:ip 产出单一 IP 形象,ip-handbook 在其上延展为完整视觉手册,确保全模块视觉风格一致。
> 调性必须继承 DESIGN.md,色彩必须来自同一调色板。

---

## 产物层目录结构

所有 ip-handbook 产出落在 `examples/ip-handbook/<project-name>/`:

```
examples/ip-handbook/<project-name>/
├── README.md                       # 手册总览(8 模块索引 + 调色板声明)
├── 01-views/                       # 模块 1:三视图
│   ├── front-line.png              # 正面线稿
│   ├── front-colored.png           # 正面填色
│   ├── side-line.png               # 侧面线稿
│   ├── side-colored.png            # 侧面填色
│   ├── back-line.png               # 背面线稿
│   └── back-colored.png            # 背面填色
├── 02-actions/                     # 模块 2:动作延展(5-8 个)
│   └── <action-slug>.png           # 如 wave.png / run.png / think.png / cheer.png
├── 03-emotions/                    # 模块 3:表情包(8-12 个)
│   └── <emotion-slug>.png          # 如 happy.png / surprised.png / shy.png
├── 04-typography/                  # 模块 4:IP 专属字体
│   ├── alphabet-en.png             # 英文字母 + 数字
│   └── alphabet-cn.png             # 中文字符
├── 05-colors/                      # 模块 5:色彩应用规范
│   └── palette.md                  # 主/辅/中性色 + 应用比例 + 场景示例
├── 06-layout/                      # 模块 6:版式要求
│   └── grid.md                     # 网格系统 + 留白 + 对齐原则
└── 07-merchandise/                 # 模块 7:周边物料
    ├── tote-bag.md                 # 帆布袋
    ├── sticker-pack.md             # 贴纸包
    ├── blind-box.md                # 限定盲盒
    └── poster.md                   # 海报物料
```

- `<project-name>` 与 [`ip`](./ip.md) 产出物的项目名对齐。
- 所有模块共享同一调色板(从 DESIGN.md 继承),模块间色彩、线条、风格 MUST 高度一致。

## 8 模块产出规格

### 模块 1:三视图

| 子项       | 规格                                                       |
| ---------- | ---------------------------------------------------------- |
| 视角数量   | 3(正面 / 侧面 / 背面)                                    |
| 每视角变体 | 2(标准线稿版 + 填色版)                                   |
| 总产出     | 6 张 PNG                                                   |
| 比例       | 3:4(2048×2732,2K 高清)                                  |
| 一致性要求 | 三视图 IP 形象特征(五官/服装/配饰)MUST 完全一致         |
| 线稿版要求 | 黑白线稿,线条粗细统一(建议 4-6px),无填色              |
| 填色版要求 | 严格使用 DESIGN.md 调色板,无渐变(扁平化)或按风格要求  |

### 模块 2:动作延展

| 子项         | 规格                                                       |
| ------------ | ---------------------------------------------------------- |
| 动作数量     | 5-8 个核心动作                                             |
| 必备动作     | 挥手 / 奔跑 / 思考 / 欢呼(4 个必备)                       |
| 选配动作     | 跳跃 / 坐下 / 指向 / 拥抱(任选 1-4 个补足 5-8 个)        |
| 每动作产出   | 1 张 PNG + 1 段动作说明(动作要点 + 适用场景)            |
| 比例         | 3:4(2048×2732,2K 高清)                                  |
| 一致性要求   | IP 形象特征(五官/服装/配色)MUST 与三视图一致             |
| 动作说明字段 | `action_name`(动作名) + `key_pose`(关键姿态) + `use_case`(适用场景) |

### 模块 3:表情包

| 子项         | 规格                                                       |
| ------------ | ---------------------------------------------------------- |
| 表情数量     | 8-12 个                                                    |
| 必备表情     | 开心 / 惊讶 / 思考 / 害羞 / 得意(5 个必备)               |
| 选配表情     | 难过 / 生气 / 困惑 / 酷 / 爱心 / 偷笑 / 困倦(任选 3-7 个)|
| 统一尺寸     | 1024×1024(正方形,便于社媒传播)                          |
| 一致性要求   | 五官风格 MUST 与三视图一致,仅表情变化                     |
| 文件命名     | `<emotion-slug>.png`(如 happy.png / surprised.png)        |
| 配色要求     | 表情可引入辅助色(如心形 #FFAA00),但主色 MUST 来自调色板 |

### 模块 4:字体设计

| 子项           | 规格                                                       |
| -------------- | ---------------------------------------------------------- |
| 字体类型       | IP 专属字体(基于 DESIGN.md typography 风格延展)          |
| 字符集         | 中文 + 英文 + 数字                                         |
| 英文 + 数字    | 26 字母(大写 + 小写)+ 10 数字(0-9),共 62 字符         |
| 中文字符       | 常用 3500 字(GB 2312 一级字库)                           |
| 字体风格       | 与 IP 形象线条风格一致(圆润/几何/手绘)                   |
| 产出格式       | PNG 预览图(字母表 + 数字表)+ 字体说明 md                |
| 字体说明字段   | `font_family`(字体族名) + `style`(风格) + `use_case`(用途) |

### 模块 5:色彩应用规范

| 子项           | 规格                                                       |
| -------------- | ---------------------------------------------------------- |
| 调色板来源     | MUST 来自 DESIGN.md frontmatter `colors`                  |
| 色彩分类       | 主色 / 辅助色 / 中性色(三类齐全)                         |
| 应用比例       | 60/30/10(主色 60% / 辅助色 30% / 中性色 10%)             |
| 场景示例       | 每类色含 3 个应用场景(如主色用于 IP 主体 / 按钮 / 强调元素) |
| 产出格式       | `palette.md`(含色卡表 + 应用比例图 + 场景示例)            |
| 色卡表字段     | `name`(色名) + `hex`(色值) + `role`(角色) + `usage`(用途) + `proportion`(占比) |
| 禁止事项       | 禁止引入 DESIGN.md 调色板之外的色值                        |

### 模块 6:版式要求

| 子项           | 规格                                                       |
| -------------- | ---------------------------------------------------------- |
| 网格系统       | 12 列网格,列宽 64px, gutter 16px                         |
| 留白规范       | 顶部/底部 64px,左右 48px,模块间 32px                     |
| 对齐原则       | 左对齐(文本)/ 居中对齐(IP 形象主体)/ 网格对齐(组合元素) |
| 产出格式       | `grid.md`(含网格示意图 + 留白规格表 + 对齐示例)            |
| 字号层级       | H1 48px / H2 36px / H3 24px / Body 16px / Caption 12px    |
| 行高           | 1.5(正文)/ 1.2(标题)                                    |

### 模块 7:周边物料(4 种)

| 物料       | 尺寸                  | 材质               | 内容要求                                         | 样机效果描述                                   |
| ---------- | --------------------- | ------------------ | ------------------------------------------------ | ---------------------------------------------- |
| 帆布袋     | 35×40cm               | 12oz 棉布          | 正面 IP 印刷 + 侧面 logo                         | 自然光下拍摄,棉布纹理可见,IP 居中印刷         |
| 贴纸包     | 单张 5×5cm,6-8 张套装 | 防水 PVC           | 每张含 1 个 IP 动作/表情 + 模切线说明            | 模切线沿 IP 轮廓,PVC 光泽质感,可贴附笔记本     |
| 限定盲盒   | 高度 7-8cm            | PVC / ABS          | 3 款造型变体 + 1 款隐藏款(隐藏款比例 1/24)     | 立体公仔实拍感,哑光质感,底座含 logo           |
| 海报物料   | A2(420×594mm),2K 高清 | 哑光铜版纸 200g    | 主视觉 IP + 辅助信息层级(标题/日期/地点)        | 远观主视觉冲击,近看辅助信息清晰,留白充足      |

- 每种物料产出 1 张样机效果 PNG + 1 份物料说明 md(含尺寸/材质/印刷工艺/内容层级)。
- 物料说明 md 字段:`name`(物料名) + `size`(尺寸) + `material`(材质) + `content`(内容) + `mockup_desc`(样机描述)。

### 模块 8:通用要求(贯穿全模块)

| 要求项         | 规格                                                       |
| -------------- | ---------------------------------------------------------- |
| 图片分辨率     | 2K 高清(2048×2732,表情包除外为 1024×1024)               |
| 比例           | 3:4(表情包除外为 1:1)                                   |
| 品牌细节       | 极致关注(IP 五官/服装/配饰/色彩在所有模块完全一致)        |
| 视觉一致性     | 色彩/线条/风格/IP 形象特征 MUST 跨模块一致                 |
| 调色板继承     | 所有模块色彩 MUST 来自 DESIGN.md `colors`                  |
| 文件命名       | kebab-case,与模块编号对齐(如 `01-views/front-line.png`)  |
| 产出索引       | `README.md` 含 8 模块索引 + 调色板声明 + 一致性声明        |

## 流程

0. **读取 DESIGN.md + ip 产出物** —— 读取项目根目录 `DESIGN.md`(调色板 + 字体)与 `examples/ip/<project-name>-ip.png`(IP 形象参考)。

   **🔴 CHECKPOINT · 输入齐全**:确认 DESIGN.md 与 IP PNG 都存在,再进入下一步。若 IP PNG 缺失,先执行 [`ip`](./ip.md) 子命令。

1. **生成模块 1 三视图** —— 基于 IP PNG 延展正面/侧面/背面,每视角线稿 + 填色版,共 6 张。

   **🔴 CHECKPOINT · 三视图一致性**:三视图 IP 特征(五官/服装/配饰)MUST 一致,再进入模块 2。

2. **生成模块 2-7** —— 按模块顺序产出动作延展 / 表情包 / 字体 / 色彩 / 版式 / 周边物料,每模块产出 PNG + 说明 md。

   **🔴 CHECKPOINT · 跨模块一致性**:每完成 1 个模块,对比前序模块的色彩/线条/IP 特征,确认一致再继续。

3. **生成模块 8 通用要求索引** —— 产出 `README.md`,含 8 模块索引 + 调色板声明(从 DESIGN.md 继承) + 一致性声明。

4. **自检** —— grep 确认所有色值来自 DESIGN.md 调色板,所有 PNG 为 2K 高清(表情包 1K),所有模块文件命名 kebab-case。

   **🔴 CHECKPOINT · 自检通过**:自检全部通过后,产出物方可交付。

## 约束汇总(硬性)

- [ ] 所有模块视觉风格 MUST 一致(色彩/线条/IP 形象特征)
- [ ] 所有色彩 MUST 来自 DESIGN.md `colors` 调色板,禁止引入新色值
- [ ] 所有图片 MUST 2K 高清(2048×2732),表情包例外为 1024×1024
- [ ] 所有图片比例 MUST 3:4,表情包例外为 1:1
- [ ] 模块 1 三视图 MUST 含 6 张(3 视角 × 2 变体)
- [ ] 模块 2 动作延展 MUST 5-8 个,含 4 必备(挥手/奔跑/思考/欢呼)
- [ ] 模块 3 表情包 MUST 8-12 个,含 5 必备(开心/惊讶/思考/害羞/得意)
- [ ] 模块 4 字体 MUST 含中英文 + 数字
- [ ] 模块 5 色彩应用比例 MUST 60/30/10(主/辅/中性)
- [ ] 模块 7 周边物料 MUST 4 种(帆布袋/贴纸包/限定盲盒/海报),每种含样机效果描述 + 尺寸 + 材质
- [ ] 产出物落在 `examples/ip-handbook/<project-name>/` 目录,按 8 模块编号子目录
- [ ] `README.md` MUST 含 8 模块索引 + 调色板声明 + 一致性声明

## 失败模式与 fallback

| 触发条件                                       | 一线修复                                                  | 仍失败兜底                                                |
| ---------------------------------------------- | --------------------------------------------------------- | --------------------------------------------------------- |
| DESIGN.md 缺失                                 | 提示用户先执行 [`design-md`](./design-md.md)              | 拒绝执行,缺 DESIGN.md 无法保证调色板一致                  |
| IP PNG 缺失(`examples/ip/` 无产出)            | 提示用户先执行 [`ip`](./ip.md) 子命令                     | 基于 DESIGN.md 调性临时构造 IP 描述,标注"无 IP PNG 参考"  |
| 三视图 IP 特征不一致(五官/服装差异)          | 重新生成,以正面图为基准对齐侧面/背面                      | 仅产出正面图,标注"侧面/背面待补"                          |
| 模块间色彩偏离 DESIGN.md 调色板                | 列出偏离色值,替换为调色板内最近色                          | 删除偏离模块,重新生成                                     |
| 表情包数量不足 8 个                            | 补足至 8 个(从选配表情补充)                              | 仅产出已有表情,标注"待补足至 8 个"                        |
| 周边物料样机效果描述空洞(缺尺寸/材质)        | 按规格表补全(帆布袋 35×40cm 12oz 棉布等)                  | 用规格表默认值,标注"默认规格"                              |
| 图片分辨率不足 2K                              | 重新生成,指定 2048×2732                                   | 接受低分辨率,标注"低清待重生成"                            |
| 字体设计字符集不全(缺中文或数字)              | 补全至中英文 + 数字齐全                                   | 仅产出英文 + 数字,标注"中文待补"                          |
| 跨模块 IP 特征漂移(如模块 3 表情风格突变)    | 以模块 1 三视图为基准,重新生成漂移模块                    | 暂停产出,提示用户确认 IP 风格基准                         |

## 禁止事项(反例)

- **禁止模块间风格不一致**(如模块 1 用扁平化,模块 2 突变 3D 渲染):风格漂移会让 IP 形象割裂,失去品牌识别度。MUST 全模块统一风格,以模块 1 三视图为基准。
- **禁止色彩脱离 DESIGN.md 调色板**(如自创主色 #FF0000):脱离调色板会破坏品牌视觉系统。MUST 所有色值来自 DESIGN.md `colors`,新色需求先更新 DESIGN.md 再用。
- **禁止跳过模块 1 三视图直接产出后续模块**:三视图是 IP 特征基准,跳过会让后续模块无对齐依据。MUST 先产模块 1,经 CHECKPOINT 一致性确认后再产后续。
- **禁止图片比例非 3:4**(表情包除外):统一比例便于手册排版与延展。MUST 3:4(表情包 1:1),严禁 16:9 / 1:1 混用(表情包外)。
- **禁止周边物料缺样机效果描述**:仅文字规格无法传达视觉感受。MUST 每种物料含样机效果描述(光线/质感/拍摄角度)。
- **禁止盲盒缺隐藏款说明**:隐藏款是盲盒核心玩法。MUST 含 3 款造型变体 + 1 款隐藏款(隐藏款比例 1/24)。
