# 图标规范

> 规范层。图标的**尺寸、风格、命名、可触达**。具体硬值在产物层 [`examples/ui-markdown/token.md`](../examples/ui-markdown/token.md)。

## 尺寸阶梯

| 用途            | 推荐尺寸 |
| --------------- | -------- |
| 导航/标签栏图标 | 24×24    |
| 列表/行内图标   | 20×20    |
| 辅助/提示图标   | 16×16    |
| 大图标/空状态   | 32×32+   |

基础栅格 24×24,图标按 24 网格设计,缩放到其他尺寸(20/16/32)时保持视觉重量一致。

## 描边 vs 填充

| 风格         | 用途               |
| ------------ | ------------------ |
| 线性(stroke) | 次功能/导航/默认态 |
| 填充(filled) | 选中态/主功能/强调 |

同一图标集 MUST 风格统一:描边粗细一致(推荐 1.5~2px stroke)、圆角一致、端点一致。**禁止混搭**(同一集里有的线性有的填充)。选中/未选中通过**同图标的线性↔填充切换**表达,不换图。

## 命名

`icon_<语义>`,语义化而非具象:

- `icon_home` ✓(语义:首页)
- `icon_house_24_blue` ✗(具象 + 尺寸 + 颜色冗余)

尺寸/颜色由容器控制,不进图标名。

## 可触达性(费茨定律)

- 可点击图标的热区 ≥ 44×44pt(视觉图标可小于 24×24,但点击热区靠 padding 撑到 44×44)
- 图标 + 文字组合优于纯图标(降低识别成本)
- 关键操作 MUST 有文字标签,不可仅用图标

## 三性

- **识别性** — 图标含义自明,小尺寸下仍可辨
- **一致性** — 整套同风格、同粗细、同栅格
- **引导性** — 配合位置/颜色引导操作,而非仅装饰

## 颜色

图标颜色**继承文字色**(同 token),不单独定义图标色。状态变化(选中/禁用)遵循文字色的透明度/色相规则,与文字一致。
