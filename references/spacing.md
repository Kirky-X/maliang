# 间距规范

> 规范层。间距的**基础单位、语义阶梯、分区策略**。具体硬值在产物层 [`examples/ui-markdown/token.md`](../examples/ui-markdown/token.md)。

## 基础单位

以 2px 为基准(4 的倍数体系,2px 仅用于微调粒度)。所有间距 MUST 是 2 的倍数(2 / 4 / 6 / 8 / 12 / 14 / 16 / 20 / 24 / 32 …)。禁止奇数或任意像素值。

## 语义阶梯

| token       | 推荐值  | 用途                   |
| ----------- | ------- | ---------------------- |
| spacing-xs  | 4       | 图标内/紧凑组内        |
| spacing-sm  | 8       | 相关元素组内           |
| spacing-md  | 14 / 16 | 模块内边距、卡片内边距 |
| spacing-lg  | 20      | 页面边距(左右安全区)   |
| spacing-xl  | 24 / 32 | 模块间距(区块之间)     |
| spacing-2xl | 48+     | 大分区/章节间距        |

## 分区策略

| 层级       | 间距         |
| ---------- | ------------ |
| 屏幕边距   | 20           |
| 模块间距   | 32(组间宽松) |
| 模块内边距 | 14(组内紧凑) |
| 组内元素   | 4~8          |

**组内紧凑、组间宽松**——邻近性法则:相关的紧挨,不相关的拉开。这是用间距表达信息分组的核心手段。

## 双倍间距技巧

图标距容器边缘采用**双倍间距**(图标内边距 > 图标间距),较大形状用两倍大圆角,视觉更平衡(见 [radius.md](./radius.md))。

## 网格

- 移动端 4 列基准网格(列宽 + 槽宽),内容对齐网格
- 页面纵向按 8px 基线网格对齐元素
