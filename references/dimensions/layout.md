# 布局规范

> 规范层。布局类型的**通用概念、选用规则、嵌套约定**。页面 markdown 的正文格式按本文件组织(章节=布局区域,可嵌套可含组件)。具体硬值(间距/圆角)在产物层 [`examples/ui-markdown/token.md`](../../examples/ui-markdown/token.md),维度规范见 [`spacing.md`](./spacing.md) / [`radius.md`](./radius.md)。
>
> 内容参考华为 HarmonyOS 布局开发文档,仅提取通用布局概念,不涉及平台代码实现。

## 总览

布局(Layout)指用特定的容器或属性管理页面中 UI 组件的大小和位置。声明式 UI 中,所有页面由组件构成,开发者按需选择合适布局进行页面开发。

### 开发流程(三步)

1. 确定页面的布局结构
2. 分析页面中的元素构成
3. 选用适合的布局容器,控制各元素的位置和大小

### 布局结构

通常为分层结构。Page 为页面根节点,容器组件(Row/Column/Stack/Flex/RelativeContainer/GridRow 等)为系统组件;不同页面结构选用不同布局容器。

### 布局元素组成(四层盒子模型)

| 层级         | 说明                                                       |
| ------------ | ---------------------------------------------------------- |
| 组件区域     | 由 width/height 设置,表示组件大小                         |
| 组件内容区   | 组件区域减去 border,作为子组件布局测算的限制              |
| 组件内容     | 组件本身占用大小(如文本);与内容区不一致时 align 属性生效 |
| 组件布局边界 | 组件区域加上 margin(外边距)                              |

### 布局分类(6 类)

| 布局类型 | 容器组件                   | 应用场景                                   |
| -------- | -------------------------- | ------------------------------------------ |
| 线性     | Row、Column                | 子元素超 1 个且能线性排列时优先            |
| 堆叠     | Stack                      | 需要堆叠效果,堆叠不影响同容器其他子组件   |
| 弹性     | Flex                       | 子组件需计算拉伸/压缩比例时优先            |
| 相对     | RelativeContainer          | 二维空间布局,页面复杂或线性嵌套过深时推荐 |
| 网格     | GridRow、GridCol           | 多设备场景辅助定位,内容相同布局不同时推荐 |
| 动态     | DynamicLayout              | 运行时切换布局算法且保持子组件状态         |

### 通用概念

- **布局容器(container)**:具有布局能力、可承载子元素并对其做尺寸计算和排列
- **布局子元素**:容器内部的元素
- **主轴(main axis)**:布局方向上的轴线,子元素默认沿主轴排列
- **交叉轴(cross axis)**:垂直于主轴方向的轴线

## 线性布局(Linear)

### 概念

开发中最常用的布局,是其他布局的基础。通过 Row(水平排列)和 Column(垂直排列)容器构建,子元素在线性方向(水平/垂直)上依次排列。排列方向由所选容器决定。

### 适用场景

布局内子元素超过 1 个,且能以某种方式线性排列时优先考虑。

### 关键属性

| 属性            | 取值                                                                   | 说明                       |
| --------------- | ---------------------------------------------------------------------- | -------------------------- |
| 方向            | Row(水平)/ Column(垂直)                                            | 由容器决定主轴方向         |
| 主轴对齐        | Start(默认)/ Center / End / SpaceBetween / SpaceAround / SpaceEvenly | 控制子元素在主轴上的分布   |
| 交叉轴对齐      | Row 取 VerticalAlign;Column 取 HorizontalAlign                        | 控制子元素在交叉轴上的对齐 |
| 单元素对齐      | alignSelf                                                              | 优先级高于容器级 alignItems |
| 间距 space      | 数值                                                                   | 排列方向上子元素的等间距   |
| 自适应          | 拉伸 / 缩放 / 延伸                                                     | 支持三种自适应能力         |

**主轴对齐语义**:
- `Start`:首端对齐(默认)
- `Center`:居中
- `End`:尾端对齐
- `SpaceBetween`:首尾贴边、中间均分
- `SpaceAround`:两端间距为中间一半
- `SpaceEvenly`:所有间距完全相等

### 嵌套规则

复杂界面多组件嵌套时,若嵌套层数过深或嵌套组件数量过多,会产生额外开销。建议通过移除冗余节点、利用布局边界减少布局计算、合理采用渲染控制语法及布局组件方法来优化性能。

## 堆叠布局(Stack)

### 概念

层叠布局在屏幕上预留一块区域显示元素,提供元素可重叠的布局。通过 Stack 容器实现位置固定定位与层叠,子元素依次入栈,**后一个子元素覆盖前一个**,子元素可叠加也可设置位置。子元素默认居中堆叠。

### 适用场景

具有较强的页面层叠、位置定位能力。典型场景:广告、卡片层叠效果;Panel 作为子组件弹出时覆盖其他组件(此时外层优先用堆叠布局)。**堆叠效果不会占用或影响同容器内其他子组件的布局空间**。

### 关键属性

| 属性           | 取值                                                                 | 说明                          |
| -------------- | -------------------------------------------------------------------- | ----------------------------- |
| 对齐 alignContent | 九宫格:TopStart/Top/TopEnd/Start/Center/End/BottomStart/Bottom/BottomEnd | 实现位置相对移动              |
| Z 序 zIndex    | 数值                                                                 | 值越大显示层级越高,大值覆盖小值 |

**覆盖规则**:若后面子元素尺寸大于前面,前面子元素完全隐藏;可通过 zIndex 调整展示。

### 嵌套规则

过多嵌套组件数会导致性能劣化。部分场景可直接使用组件属性或借助系统 API 替代层叠布局效果,以减少嵌套组件数、优化性能。

## 弹性布局(Flex)

### 概念

弹性布局提供更有效的方式对容器中子元素进行排列、对齐和分配剩余空间。容器默认存在主轴与交叉轴,子元素默认沿主轴排列。子元素在主轴方向的尺寸称主轴尺寸,在交叉轴方向称交叉轴尺寸。

### 适用场景

页面头部导航栏均匀分布、页面框架搭建、多行数据排列等;在子组件需要计算拉伸或压缩比例时优先使用,可使多个容器内子组件有更好视觉填充效果。

### 关键属性

| 属性            | 取值                                                                   | 说明                       |
| --------------- | ---------------------------------------------------------------------- | -------------------------- |
| 布局方向 direction | Row(默认,水平起始端)/ RowReverse / Column / ColumnReverse          | 控制主轴方向与起止         |
| 布局换行 wrap   | NoWrap(默认,不换行)/ Wrap / WrapReverse                            | 子元素宽度总和超出时的行为 |
| 主轴对齐        | Start(默认)/ Center / End / SpaceBetween / SpaceAround / SpaceEvenly | 语义同线性布局             |
| 交叉轴对齐 alignItems | Auto(默认)/ Start / Center / End / Stretch / Baseline            | Stretch 拉伸填充;Baseline 文本基线 |
| 子元素交叉轴对齐 | 单独设置                                                               | 优先级高于容器设置         |
| 自适应拉伸      | 支持                                                                   |                            |

**NoWrap 行为**:子元素宽度总和超出容器时不换行,被子元素被压缩。

### 与线性布局的差异

| 维度       | 线性布局         | 弹性布局                 |
| ---------- | ---------------- | ------------------------ |
| 子组件压缩 | 不能             | 默认能够压缩或拉伸       |
| 换行       | 单方向排列       | 支持多行布局             |
| 适用侧重   | 基础首选         | 需拉伸/压缩比例时优先    |

### 嵌套规则

文档未单独明确弹性布局的嵌套约定,默认遵循通用嵌套性能建议(避免层数过深)。

## 相对布局(Relative)

### 概念

相对布局是在**二维空间**中的布局方式,不需要遵循线性布局规则,布局方式更自由。通过在子组件上设置锚点规则(AlignRules),使子组件能将自己在横轴、纵轴中的位置与容器或容器内其他子组件对齐。锚点规则天然支持子元素压缩、拉伸、堆叠或形成多行效果。

### 适用场景

页面元素分布复杂,或通过线性布局会使容器嵌套层数过深时推荐使用;适用于处理复杂界面、对多个子元素进行对齐和排列。

### 关键属性

| 属性            | 取值                                                                   | 说明                                  |
| --------------- | ---------------------------------------------------------------------- | ------------------------------------- |
| 锚点 anchor     | 父容器(`__container__`,默认)/ 兄弟元素 / 辅助线 guideline / 屏障 barrier | 设置当前元素基于哪个元素确定位置      |
| 参考边界(水平) | left / middle / right                                                  | 三者同设时仅 left 和 middle 生效      |
| 参考边界(垂直) | top / center / bottom                                                  | 三者同设时仅 top 和 center 生效       |
| 对齐位置 align  | HorizontalAlign.Start/Center/End;VerticalAlign.Top/Center/Bottom      | alignRules 的 align 字段              |
| 依赖关系        | id 唯一                                                                 | 未设 id 可显示但无法被引用为锚点;**必须避免依赖循环** |
| 位置偏移        | offset(x/y) / bias                                                     | API 11+ 推荐用 bias 设置额外偏移      |
| 链 chain        | 首尾相连                                                               | 将一系列组件对齐,可指定排列方式      |
| 辅助线 guideline | 容器内虚拟锚点                                                         | 便于统一对齐至某个偏移位置            |
| 屏障 barrier    | 一组组件在特定方向上的共同最远边界                                     | 如一组组件底部边缘中最底部的边界      |

**优先级**:id 冲突时遵循 组件 > guideline > barrier。

**offset 注意**:以 offset 调整位置的组件作为锚点时,对齐位置为设 offset 之前的位置。

### 嵌套规则

Row、Column、Flex、Stack 等多种布局组件均可按 RelativeContainer 规则进行对齐排布(即相对布局可作为外层容器,内部嵌套其他布局组件并通过 alignRules 定位)。

## 网格布局(Grid)

> 采用**栅格布局(GridRow/GridCol)**语义:基于断点的响应式栅格系统,核心是「断点 + 列数 + 跨度」。栅格不同于网格布局固定的空间划分,可实现不同设备下不同布局。

### 概念

栅格布局是一种通用的辅助定位工具,将空间分割为有规律的栅格(等宽列数和行数)。主要优势:① 提供可循规律,解决多尺寸多设备动态布局问题;② 统一定位标注,保证多设备布局一致性;③ 灵活间距调整;④ 自动换行和自适应。

### 适用场景

多设备场景下通用辅助定位;内容相同但布局不同时使用;一次开发多端适配。

### 关键属性

| 属性                  | 取值                                                                   | 说明                                  |
| --------------------- | ---------------------------------------------------------------------- | ------------------------------------- |
| 断点 breakpoints      | 默认 4 个:xs [0,320) / sm [320,600) / md [600,840) / lg [840,+∞)    | 最多支持 6 个(可启用 xl、xxl)       |
| 断点自定义            | 单调递增数组(最大长度 5)                                            | 自定义断点位置                        |
| 断点参考物 reference  | 窗口尺寸等                                                             | 应用以非全屏窗口显示时,以应用窗口宽度为参照更通用 |
| 总列数 columns        | number(所有设备同一列数)或 GridRowColumnOption(按断点分别配置)   | API 20 之前默认 12;API 20+ 默认 {xs:2,sm:4,md:8,lg:12,xl:12,xxl:12} |
| 排列方向 direction    | Row(默认,从左往右)/ RowReverse(从右往左)                          |                                       |
| 子组件间距 gutter     | number(水平垂直等距)或 {x, y}(分别设置)                           |                                       |
| 子组件 span           | number 或按断点配置                                                    | 占列数,决定宽度,默认 1              |
| 子组件 offset         | number 或按断点配置                                                    | 偏移列数                              |
| 子组件 order          | number 或按断点配置                                                    | 元素序号                              |
| 断点回调 onBreakpointChange | 监听器                                                           | 监听断点切换                          |

### 嵌套规则

栅格组件支持嵌套使用(具体规则见平台文档)。栅格布局可作为外层容器,内部 GridCol 可嵌套其他布局组件。

## 动态布局(Dynamic)

> 涵盖两层语义:(a) 响应式通用策略(拉伸/缩放/占比/隐藏);(b) 运行时布局算法切换(DynamicLayout 组件)。

### 概念(响应式通用策略)

容器尺寸变化时,对子元素的约束策略(来自布局总览):

| 策略 | 说明                                                   | 控制方式                                   |
| ---- | ------------------------------------------------------ | ------------------------------------------ |
| 拉伸 | 容器尺寸变化时,增减空间全部分配给指定区域             | flexGrow(基于剩余空间拉伸)/ flexShrink(按压缩尺寸压缩) |
| 缩放 | 子组件宽高按预设比例随容器变化,宽高比不变             | aspectRatio(= width/height)               |
| 占比 | 子组件宽高按预设比例随祖先容器变化                     | ① 百分比宽高(参照父/祖先组件宽高);② layoutWeight(自适应占满剩余空间) |
| 隐藏 | 按预设显示优先级,随容器尺寸变化显示/隐藏              | displayPriority(相同优先级同时显隐)       |

### 概念(DynamicLayout 组件)

DynamicLayout 是动态布局容器组件,支持在运行时**动态切换不同的布局算法**,同时**保持子组件状态不变**(如输入框内容、滚动位置等)。通过它可灵活实现同一内容在不同场景下的多种布局展示。

### 适用场景

- 响应式布局(如横竖屏切换)
- UI 页面需在运行时切换为不同布局效果(列表、网格、堆叠等)
- 需要实现瀑布流、标签云等自定义不规则布局效果时,使用其自定义布局能力

### 关键策略

**内置布局算法**:

| 算法                   | 等效容器 | 支持属性                                            |
| ---------------------- | -------- | --------------------------------------------------- |
| RowLayoutAlgorithm     | Row      | space、justifyContent、alignItems、isReverse        |
| ColumnLayoutAlgorithm  | Column   | space、justifyContent、alignItems、isReverse        |
| StackLayoutAlgorithm   | Stack    | alignContent(九宫格);子组件可用 layoutGravity 单独设置 |
| GridLayoutAlgorithm    | Grid     | columnsTemplate、行间距/列间距、align               |

**自定义布局算法**(CustomLayoutAlgorithm):继承该类,重写 `onMeasure`(自定义测量子组件大小)和 `onLayout`(自定义排列子组件位置)方法,实现个性化布局(瀑布流、标签云等)。

**切换布局算法**:运行时切换算法,子组件状态保持不变。

### 约束与限制

1. 布局算法类使用 `@ObservedV2` 装饰,不支持 `@State` 装饰器
2. 切换布局算法时,子组件状态(输入框内容、滚动位置等)保持不变
3. 自定义算法的 `onMeasure`/`onLayout` 中**不允许修改状态变量**,避免不可预期行为

### 嵌套规则

文档未明确说明 DynamicLayout 的嵌套约定,默认遵循通用嵌套性能建议。

## 布局选用决策表

| 场景                                         | 推荐布局               | 理由                                                       |
| -------------------------------------------- | ---------------------- | ---------------------------------------------------------- |
| 子元素超 1 个,能水平/垂直线性排列            | 线性(Row/Column)     | 最常用、最基础,开销低,优先考虑                           |
| 需要元素重叠/堆叠(广告、卡片层叠、Panel 弹出) | 堆叠(Stack)           | 提供重叠能力,堆叠不影响同容器其他子组件空间               |
| 子组件需要按比例拉伸/压缩填充空间            | 弹性(Flex)            | 默认支持子组件压缩/拉伸,视觉填充效果好                    |
| 多行数据排列、导航栏均匀分布                 | 弹性(Flex)            | 支持换行和剩余空间分配                                     |
| 页面元素分布复杂,二维自由定位               | 相对(RelativeContainer) | 二维空间布局,不遵循线性规则,锚点自由                      |
| 线性布局导致嵌套层数过深                     | 相对(RelativeContainer) | 减少嵌套深度,提升性能                                     |
| 多设备(手机/平板/折叠屏)同一内容不同布局    | 网格(GridRow/GridCol) | 断点 + 列数自适应,一次开发多端适配                         |
| 运行时切换布局(如横竖屏列表↔网格)且保持状态 | 动态(DynamicLayout)   | 动态切换算法,子组件状态不变                               |
| 瀑布流、标签云等不规则布局                   | 动态(DynamicLayout)+ 自定义算法 | CustomLayoutAlgorithm 支持 onMeasure/onLayout 自定义 |
| 根据容器尺寸做差异化组件创建                 | 容器断点(ContainerReader) | 双向绑定实时返回容器尺寸和断点                            |
| 单页面内视图内容快速切换                     | 选项卡(Tabs)          | 提升查找效率,精简单次信息量                              |
| 元素需相对原位置小幅偏移(不脱离文档流)      | 相对定位(offset)      | 原位置保留,不影响父容器布局                               |
| 元素需相对父容器固定偏移                     | 绝对定位(position)    | 左上角偏移,但屏幕适配较差                                 |

## 页面 markdown 的布局表达

`examples/ui-markdown/` 页面 markdown 的正文格式按本文件组织。约定如下:

### 章节即布局区域

页面 markdown 的**正文按从上到下、从左到右的视觉顺序组织章节**,每个 `##` 章节对应页面一个布局区域。

### 章节支持嵌套布局(布局嵌套布局)

章节内可用 `###` 子章节进一步划分布局区域,表达**布局嵌套布局**。例如:

```
## 3. 金刚区入口          # 外层布局区域
### 3.1 item 字段结构     # 内层嵌套布局
```

### 章节支持直接含组件(布局含组件)

章节内可直接用**组件参数表**描述,表达**布局直接包含组件**。例如:

```
## 2. 搜索栏

| 参数     | 值                              |
| -------- | ------------------------------- |
| 宽度     | match-parent − 2×`{spacing-lg}` |
| 高度     | 36px                            |
| ...      | ...                             |
```

### 章节顺序约定

- **第一章 = 顶部导航**(引用 [`organisms/nav-bar.md`](../../examples/ui-markdown/organisms/nav-bar.md))
- 中间章节 = 页面主体各区块(每区块一章)
- **最后一章 = 底部 dock**(引用 [`organisms/dock.md`](../../examples/ui-markdown/organisms/dock.md),仅一级页面含 dock;二级页面不含)

### 章节内子组件顺序

每个章节内,子组件**从左到右**描述;内部组件用嵌套子章节展开。

### 格式示例

参见 [`examples/ui-markdown/ui/home.md`](../../examples/ui-markdown/ui/home.md)(一级页面,含 5 个布局章节)与 [`examples/ui-markdown/ui/setting/about.md`](../../examples/ui-markdown/ui/setting/about.md)(二级页面,无 dock,含嵌套子章节)。

## 嵌套规则与性能

### 避免嵌套层数过深

复杂界面多组件嵌套时,若嵌套层数过深或嵌套组件数量过多,会产生**额外开销**。建议:

- 移除冗余节点(无实际布局意义的中间容器)
- 利用布局边界(margin)减少布局计算
- 合理采用渲染控制语法及布局组件方法优化性能

### 何时用相对布局替代深层线性嵌套

当线性布局导致容器嵌套层数过深时,推荐改用**相对布局(RelativeContainer)**:相对布局在二维空间布局,不需要遵循线性规则,可通过锚点自由定位,显著减少嵌套深度。

### 堆叠布局的嵌套优化

部分场景可直接使用组件属性或借助系统 API 替代层叠布局效果,以减少嵌套组件数、优化性能。

## 响应式策略

### 通用策略(对子元素的约束)

| 策略 | 控制方式                                   | 适用场景                       |
| ---- | ------------------------------------------ | ------------------------------ |
| 拉伸 | flexGrow / flexShrink                      | 容器增减空间全部分配给指定区域 |
| 缩放 | aspectRatio                                | 宽高比不变的等比缩放           |
| 占比 | 百分比宽高 / layoutWeight                  | 按预设比例随祖先容器变化       |
| 隐藏 | displayPriority                            | 按显示优先级随容器尺寸显隐     |

### 断点与栅格

- **断点**:以设备水平宽度(vp 单位)为依据划分设备宽度类型(默认 xs/sm/md/lg,最多 6 个)
- **栅格**:断点 + 总列数 + 子组件跨度(span/offset/order),实现多设备布局一致性
- **断点参考物**:应用以非全屏窗口显示时,以应用窗口宽度为参照更通用
- **断点回调**:onBreakpointChange 监听断点切换

### 动态布局(运行时切换)

- 内置算法:Row/Column/Stack/Grid 四种,与对应容器效果一致
- 自定义算法:CustomLayoutAlgorithm 重写 onMeasure/onLayout,实现瀑布流、标签云等不规则布局
- **状态保持**:切换布局算法时,子组件状态(输入框内容、滚动位置等)保持不变
- **约束**:自定义算法的 onMeasure/onLayout 中不允许修改状态变量

## 定位

### 绝对定位(position)

- **概念**:元素左上角相对父容器左上角偏移
- **文档流**:**脱离文档流**
- **影响**:不影响父容器布局,仅绘制时调整
- **权衡**:屏幕适配较差

### 相对定位(offset)

- **概念**:相对自身原位置偏移
- **文档流**:**不脱离文档流**,原位置保留
- **影响**:不影响父容器布局,仅绘制时调整
- **适用**:元素需相对原位置小幅偏移的场景

### 选用建议

| 需求                                 | 推荐定位     |
| ------------------------------------ | ------------ |
| 元素相对父容器固定偏移               | 绝对定位(position) |
| 元素相对自身原位置小幅偏移           | 相对定位(offset)   |
| 需要保持文档流原位置                 | 相对定位(offset)   |
| 屏幕适配要求高                       | 避免绝对定位 |
