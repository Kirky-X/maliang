# 颜色规范

> 规范层。颜色的**使用规则**(原则)。192 产品类型的具体调色板查询在 [`color-palettes.md`](./color-palettes.md);最终 RGBA+HEX 硬值在产物层 [`examples/ui-markdown/token.md`](../../examples/ui-markdown/token.md)。本文件不含具体色值,只讲规则。

## 1. 60-30-10 配色比例

- **主色 60%** — 大面积背景/容器
- **辅助色 30%** — 次要表面/分区/卡片
- **强调色 10%** — CTA/关键操作/焦点状态

明确三色角色,避免某种颜色过突出或压抑其他。三色比例失衡是界面"花"的首要原因。

## 2. 中性色平衡

中性色(白/灰/黑梯度)承担 60-30 主体,给品牌色留出 10% 强调空间。中性色按明度梯度**成体系**(如 50..900 九阶),而非零散取值。中性梯度不稳,品牌色再准也压不住。

## 3. 品牌色扩展

以单一品牌色为根,通过调明度/饱和度派生完整梯度(brand-50..brand-900)。不要为不同状态引入第二种品牌色——状态靠**同色相的明度/饱和变化**表达,不靠换色相。

## 4. 避免纯黑/纯白

- 文字最深 ≠ 纯黑,用极深灰(接近黑、非纯黑),降低对比刺眼感
- 背景最亮 ≠ 纯白,用极浅灰(接近白、非纯白),减轻视觉疲劳

纯黑/纯白仅用于刻意的高对比场景(如 OLED 纯黑底、纯白摄影底)。

## 5. 暗色模式独立设计

暗色**不是**亮色的简单反色。暗色模式下:

- 背景层级靠**提亮**(深灰梯度,非纯黑叠加),用表面 elevation 区分层而非靠纯黑
- 文字靠**降饱和**(纯白刺眼,用浅灰)
- 强调色饱和度适当下调,避免暗底过刺;高饱和色仅小面积点缀

语义 token 通过主题映射切换明暗,而非每处硬改。

## 6. 功能色(语义固定)

| 语义色  | 角色            | 约束               |
| ------- | --------------- | ------------------ |
| success | 成功/正向       | 跨页面永远同一色相 |
| warning | 警示/提醒       | 跨页面永远同一色相 |
| danger  | 错误/破坏性操作 | 跨页面永远同一色相 |
| info    | 中性提示        | 跨页面永远同一色相 |
| link    | 链接/可点击     | 跨页面永远同一色相 |

功能色语义全局一致:success 永远绿、danger 永远红,不因页面换色。用户靠色相识别语义,换色相即破坏识别。

## 7. 状态色(叠加在功能色上)

| 状态     | 表达                      |
| -------- | ------------------------- |
| default  | 静止基线                  |
| hover    | 悬停(轻微提亮/加深)       |
| pressed  | 按下(反向微调)            |
| focused  | 键盘聚焦(常配 focus ring) |
| disabled | 不可用(降透明度,非换灰)   |
| selected | 选中(提饱和或加底色)      |

状态色是功能色基础上的明度/饱和微调,**不引入新色相**。disabled 统一降透明度而非换灰色,保留语义可读性。

## 8. WCAG 对比度强制

> 本节为颜色相关的对比度硬性规则。完整 WCAG 规则(含 prefers-* 偏好)见 [`accessibility.md`](../meta/accessibility.md)。Accessibility 为 CRITICAL 优先级(见 [`rules-priority.md`](../meta/rules-priority.md)),不可被任何颜色美学覆盖。

### 阈值表

| 文本类型             | AA(最低) | AAA(目标) |
| -------------------- | --------- | ----------- |
| 正文(< 18pt / 14pt bold) | 4.5:1     | 7:1         |
| 大文本(≥ 18pt / 14pt bold) | 3:1       | 4.5:1       |
| 非文本(UI 组件 / 图形) | 3:1       | —           |

### 颜色配对强制项

- `color-text-primary` 与 `color-bg-primary` 配对**必须**通过 AA(4.5:1 正文)
- `color-text-secondary` 即使作为辅助文字,只要承载信息,也必须 ≥ 4.5:1
- `color-text-disabled` 不豁免对比度,需 ≥ 3:1(保证可识别)
- `color-border` 与背景对比度 ≥ 3:1(用于表单 / 输入框边框识别)
- `placeholder` 文字 ≥ 4.5:1,不可用 `text-gray-300` 凑数

### 状态色对比度

| 状态     | 对比度规则                                       |
| -------- | ------------------------------------------------ |
| hover    | 不可降低对比度(提亮通常降低对比度,需校验)     |
| pressed  | 不可低于 default 的 90%                          |
| focused  | focus ring 与背景对比度 ≥ 3:1                    |
| disabled | 文本对比度仍 ≥ 3:1(不豁免)                    |

### 暗色模式对比度

暗色模式下对比度更易不达标(浅灰文字 on 深灰背景):

- 暗色背景的 `color-text-primary` 不可用纯白(`#fff`),但对比度必须 ≥ 4.5:1 → 用 `#f7f5f2` 或 `#e5e5e5` 一类浅灰
- 暗色强调色饱和度下调,但对比度仍需 ≥ 3:1(大文本 / UI 组件)
- 暗色模式下应优先 AAA(7:1),因暗色背景上视觉疲劳更明显

### 功能色对比度

功能色(success/warning/danger/info)作为信息载体时:

- success 绿色 on 白底 ≥ 4.5:1
- warning 黄色 on 白底常不达标(需用深一档 amber / orange,而非 yellow)
- danger 红色 on 白底 ≥ 4.5:1
- info 与品牌色同源,继承品牌色对比度

### 自动校验

- design-md Phase 3 Lint 阶段:`npx @google/design.md lint` 自动计算每个组件 `backgroundColor` / `textColor` 配对的 WCAG 对比度,低于 AA 触发 warning
- preview 阶段 Pre-Flight Check:扫描所有文字与背景配对,违反 = 硬性失败
- 工具:WebAIM Contrast Checker / Chrome DevTools Contrast Ratio 检查器

## 与调色板/产物层的关系

```
选调色板(color-palettes.md,按产品类型查 192 套)
  → 把 Primary/Secondary/Accent 映射到本规范的 60-30-10 角色
  → 派生中性梯度 + 功能色 + 状态色
  → 落为产物层 examples/ui-markdown/token.md 的 RGBA+HEX 硬值
```

本规范管"怎么用色",调色板管"用哪套色",产物层管"色的精确值"。
